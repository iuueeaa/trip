## 画像が表示されない場合の確認順序

1. ファイルが実在するか (ls / file_exists)
2. パスが正しいか (絶対/相対、URL/ファイルパス)
3. 拡張子の大文字小文字
4. ファイルパーミッション(644相当か)
5. .htaccess / nginx設定でブロックされていないか
6. CSS で display:none / visibility:hidden になっていないか
7. img要素の src 属性が正しく出力されているか
8. ブラウザの開発者ツールでネットワークエラー確認

## CSS/JSが読み込まれない場合の確認順序

1. ファイルが実在するか
2. enqueue / register で正しく登録されているか
3. パスが正しいか(get_template_directory_uri 等)
4. キャッシュが効いていないか
5. ビルドが走っているか(\_src → \_public)
6. ブラウザの開発者ツールで404確認

## include/require が失敗する場合

1. ファイルが実在するか
2. パスの解決(相対パスの基準ディレクトリ)
3. 大文字小文字
4. ファイルパーミッション
5. 構文エラーで読み込みに失敗していないか
