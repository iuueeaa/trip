## 運用ルール

### レビューポリシー

- **タイミング**: TASK 単位（1 TASK 完了 → Opus レビュー → OK → 次 TASK）
- **基準**: `docs/checklist-review.md` の Opus 担当セクション全項目
- **完了条件**: 致命的・要修正ゼロ。軽微のみ残存は完了可
- **大規模 TASK**: 複数日かかるものはダブルレビュー（Opus + GPT 反論）

### チャット新設ルール

- **推奨タイミング**: Phase 単位（1-A 完了 → 新チャット → 1-B 開始）
- **5 TASK を超えたら**: チャットが長くなるサインなので新設を検討
- **新チャット開始時**: `CLAUDE.md` の「よく使うプロンプト > 新チャット開始時」を使う
- **理由**: 長いチャットでは `CLAUDE.md` ルールより会話履歴の重みが増し、指示が薄れる
